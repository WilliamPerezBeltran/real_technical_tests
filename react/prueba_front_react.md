# Prueba

Usando la [API de Gists de GitHub](https://developer.github.com/v3/gists/), debes desarrollar una aplicación para consultar gists públicos.

## Requerimientos

1. Listar y paginar los gists públicos de GitHub  
   - Cada elemento de la lista debe incluir:  
     - Nombre e imagen del propietario  
     - Descripción  
     - Fecha de creación  
   - La paginación debe mostrar **20 gists por página**  

2. Debe ser posible ver el detalle de un gist:  
   - Propietario con nombre e imagen  
   - Descripción  
   - Fecha de creación  
   - Enlace a GitHub con el ID del gist  
   - Listado de enlaces a los archivos con sus nombres  

3. Debe ser posible regresar al listado de gists.  
4. Si no existe propietario, debe mostrarse **Anonymous** y una imagen de *placeholder*.  

## Extras (sin orden de importancia)

- Dividir la aplicación en varios componentes reutilizables  
- Utilizar técnicas de división de código (code splitting)  
- Documentar el código  
- Usar **Router**  
- Usar **Redux**  
- Usar **ES6, ES2015**, etc.  
- Usar **styled-components**  

## Condiciones de juego

- Debe desarrollarse con **ReactJS**  
- Puedes usar un *boilerplate*  
- Puedes consultar lo que desees en internet, pero **no copiar código de otros**  
- Todo debe hacerse con **componentes funcionales** y **hooks**  

## Recomendaciones

- Puedes usar la librería que desees para realizar las peticiones **AJAX**  
- Solo se probará en las últimas versiones de **Chrome**  
- Si tienes dudas, puedes preguntar en cualquier momento  

## ¿Qué se evaluará?

En orden de importancia:

1. No haber incumplido las condiciones de juego  
2. Funcionamiento (es más importante que funcione a que se vea bonito)  
3. Orden y organización del código  
4. Extras  

## Por último (pero no menos importante)

Crea un **repositorio público** en tu cuenta de **GitHub** o **Bitbucket** con el nombre  
**`github-test-react`** y compártelo con nosotros para poder ver y probar tu código.
